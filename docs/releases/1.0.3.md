# Markdown Core 1.0.3

This maintenance release improves repository setup, cross-platform validation,
and release reliability. Public parsing and canonical AST APIs are unchanged.

- A single environment entry point now checks or prepares the pinned tools used
  by contributors, CI, IDE imports, package audits, and release staging.
- Cross-platform CI uses isolated push, pull-request, and merge-queue lanes with
  stable, fail-closed quality gates.
- C, SwiftPM, Kotlin Multiplatform, Android, JVM, npm, and WebAssembly package
  consumers receive stronger staged-artifact and public-surface validation.
- Release staging and recovery preserve the validated artifact bytes and avoid
  coupling tag releases to historical branch check runs.
- The repository includes a reusable setup template documenting platform-native
  binding practices, protected release configuration, and operational lessons.
