# Markdown Core 1.0.2

This patch improves Kotlin/JVM native-library lifecycle management and the
Kotlin Multiplatform development experience. Public parsing and canonical AST
APIs are unchanged.

- Kotlin/JVM applications no longer accumulate empty `markdown-core-*`
  directories after a clean shutdown. The bundled JNI library and its temporary
  directory are now registered for deletion in the order required by the JVM.
- The JVM loader now uses the platform's native-library name mapping and avoids
  overwriting an unexpected path while retaining zero-configuration loading
  from the published JAR.
- Android Studio and IntelliJ IDEA retain the Kotlin Multiplatform source sets
  after Gradle sync, including common, JVM, Android, and Native sources.
- The repository provides an `All Kotlin tests` Gradle entry for running every
  test suite supported by the current host, including both Android managed
  device page-size configurations.
- Consumer documentation now focuses on installation, parsing, traversal,
  platform support, and runtime requirements.
