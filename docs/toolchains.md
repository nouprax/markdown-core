# Toolchains and IDE Support

This document records the tool versions selected for the initial Markdown Core
monorepo setup. Upgrades must be reviewed together with lockfiles, Gradle
verification metadata, formatter output, and the full validation matrix.

## Supported IDEs

- Android Studio 2026.1.1, build `AI-261.23567.138.2611.15646644`, with Gradle
  configured to use the repository-provisioned JDK 26 toolchain.
- IntelliJ IDEA 2026.1 stable with JDK 26 for the Gradle daemon.

A clean import starts at the repository root. It must not require release
credentials, signing keys, generated `.idea`/`.iml` files, or prebuilt native
libraries. Android Studio may discover the SDK through its normal configuration;
command-line builds use `ANDROID_HOME` or the standard macOS SDK location.

Before release, perform a clean sync in both supported IDEs. The headless model
smoke used in CI is:

```sh
scripts/gradle-model-smoke.sh
```

## Gradle policy

- Plugin and dependency repositories are centralized in `settings.gradle.kts`.
- Dependency locking is enabled for resolvable project configurations.
- `gradle/verification-metadata.xml` records checksums and available signatures.
- Configuration cache, parallel configuration cache, local build cache, and
  parallel execution are enabled in `gradle.properties`.
- Toolchain download URLs are generated by Gradle's `updateDaemonJvm` task using
  the Foojay resolver plugin.
- Release credentials are not read during configuration or IDE sync.

Project dependencies use `project.dependencies.project(...)`; the deprecated
Project-object dependency notation is forbidden because Gradle 10 removes it.
The repository declaration no longer passes a `Project` object. A remaining
warning is emitted by AGP 9.2.1 while creating the KMP Android host-test
component (`VariantDependencies.createForKotlinMultiplatform`). Android's AGP
9.3 release notes record the upstream migration away from Project dependency
notation, but 9.3 is not yet stable. Phase 20 must adopt AGP 9.3 stable before
the first release and run cache-cold model, host-test, device-test, publication,
and consumer checks with `--warning-mode=fail`. Phase 17 does not adopt a preview
AGP merely to hide a warning whose repository-owned source has already been
removed.

## Common commands

```sh
npx --yes pnpm@11.7.0 install --frozen-lockfile
npx --yes pnpm@11.7.0 format:check
npx --yes pnpm@11.7.0 lint
scripts/gradle.sh projects
scripts/gradle.sh ktlintCheck
scripts/lint-c.sh
```

Android emulator configuration is repository-owned. Gradle Managed Devices
provisions the declared Pixel 10 Pro XL/64-bit Google APIs images at API 36 with
both 4 KB and 16 KB page sizes, then runs the two separate
native instrumentation selections:

```sh
pnpm test:kotlin-android-emulator
pnpm conformance:kotlin-android-emulator
```

Neither command reads an Android Studio AVD or device serial. The first run may
download the declared system image into the Android SDK cache. GMD stops the
emulators and restores clean snapshots automatically while retaining reusable
AVD/snapshot caches. Reclaim those managed AVD caches explicitly when needed:

```sh
pnpm clean:kotlin-android-emulator
```

This maintenance task is intentionally not attached to correctness or
conformance; the shared SDK system-image packages remain installed. Linux x64 native
execution is intentionally CI-only: the `ubuntu-latest` Kotlin matrix entry
calls `test:kotlin-linux-x64` and `conformance:kotlin-linux-x64` directly. The
repository does not provide a local container or emulation substitute for that
required-CI evidence.

When adding or upgrading a Gradle plugin or dependency, regenerate metadata in
a dedicated reviewable change:

```sh
scripts/gradle.sh help ktlintCheck \
    :packages:kotlin-markdown-core:dependencies \
    --write-locks \
    --write-verification-metadata sha256,pgp
```

Review every new repository, component, checksum, trusted key, and ignored key;
never accept verification metadata changes as an unreviewed build by-product.
