# Phase 11: Swift immutable AST binding

Phase 11 introduces the `MarkdownCore` SwiftPM library product. The public
module owns a fully copied, immutable Swift value tree and does not expose or
retain native document or node handles.

> Phase 18 replaces the package-local expected tree literal with the sole root
> canonical AST manifest. macOS and iOS Simulator conformance package that same
> source as a SwiftPM test resource and still dump only the public Swift tree.

## Public contract

- `Document.parse(_:options:)` is the single synchronous parse entry point.
- `ParseOptions` defaults match the frozen cross-platform feature set.
- Every public node conforms to `Markup` and `Sendable`, has a complete
  non-optional `Scope`, immutable behavior-bearing fields, and immutable child
  values.
- `MarkupVisitor` requires one typed overload for every concrete `Markup` and
  deliberately provides no catch-all or default implementation.
- `Walker` emits read-only depth-first entering and exiting events.
- Public `TreeDumper.dump(markup)` and `Markup.dump()` emit the canonical
  diagnostic tree for a document or subtree.
- Parsing recursively copies every scope, string, optional value, enum, and
  child into Swift before releasing the native document.

The C implementation is an internal SwiftPM target (`MarkdownCoreC`) and is not
a library product. Consumers depend only on the `MarkdownCore` product and
write `import MarkdownCore`.

## Verification

Swift Testing suites cover API shape, focused canonical AST snapshots, Unicode,
failure semantics, ownership, visitor/walker traversal, and `Sendable` use.
Canonical text is generated by independently traversing the Swift value tree;
the C diagnostic dump and C-owned goldens are never read. Public `TreeDumper`
uses exhaustive `MarkupVisitor` dispatch for record formatting and `Walker`
enter/exit events for hierarchy; the `Markup.dump()` default implementation
delegates to it for any subtree. A nested clean SwiftPM consumer verifies
product-only use.

Correctness, conformance, and performance use separate native targets.
`test:swift-macos`/`test:swift-ios-simulator` select `MarkdownCoreTests`;
the matching `conformance:swift-*` tasks select
`MarkdownCoreConformanceTests`; `benchmark:swift-macos` delegates the release
benchmark executable. CI builds the package contract for iOS 18/26 and macOS
15/26 with Xcode 26.
